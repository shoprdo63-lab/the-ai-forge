# VRAM Budgeting for Multimodal Models

Multimodal models present unique challenges for memory management. Understanding VRAM requirements is crucial for optimal performance.

## Model Memory Analysis

### Vision-Language Models
- **LLaVA**: 12GB minimum for 7B model
- **CLIP**: 8GB for large variants
- **BLIP-2**: 16GB for Flan-T5 XXL

### Video Understanding
- **Video-ChatGPT**: 24GB for 4K processing
- **InternVideo**: 16GB for 1K resolution
- **VideoMAE**: 12GB for standard models

## Optimization Strategies

### Quantization Techniques
```python
# 4-bit quantization reduces VRAM by 75%
from transformers import BitsAndBytesConfig

quantization_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_use_double_quant=True,
)
```

### Gradient Checkpointing
```python
# Trade compute for memory
model.gradient_checkpointing_enable()
```

### Memory Mapping
```python
# Efficient memory allocation
torch.cuda.empty_cache()
```

## Memory Profiling

```python
import torch
import psutil

def check_memory():
    gpu_memory = torch.cuda.memory_allocated() / 1024**3
    cpu_memory = psutil.virtual_memory().percent
    print(f"GPU: {gpu_memory:.1f}GB, CPU: {cpu_memory}%")
```
